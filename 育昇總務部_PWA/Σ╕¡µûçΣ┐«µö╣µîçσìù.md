# 中文修改指南 - 哪些地方可以自己改？

這份文件會告訴你專案中哪些地方可以修改，以及如何修改成你想要的樣子。

## 📱 介面文字修改

### 1. PWA 應用程式名稱
**檔案位置**：`client/public/manifest.json`

```json
{
  "name": "iPhone Notes PWA",           // ← 改成你想要的完整名稱
  "short_name": "備忘錄",               // ← 改成你想要的簡短名稱（顯示在手機桌面）
  "description": "iPhone 風格的電子記事本 PWA"  // ← 改成你的描述
}
```

**範例**：
```json
{
  "name": "我的個人筆記本",
  "short_name": "筆記",
  "description": "用來備份 iPhone 備忘錄的工具"
}
```

---

**檔案位置**：`client/src/pages/SettingsLocal.tsx`

找到這一段（大約第 72 行）：
```typescript
await notesDB.addFolder({
```

**可用的 emoji 範例**：
- 📁 資料夾
- 📝 筆記
- 💼 工作
- 🏠 家庭
- 💡 想法
- 📚 學習
- ❤️ 重要

## 🎨 外觀樣式修改

### 6. 主題顏色
**檔案位置**：`client/src/index.css`

找到這些變數（大約第 8-30 行）：

```css
:root {
  --background: oklch(1 0 0);        /* 背景色（白色） */
  --foreground: oklch(0.145 0 0);    /* 文字色（黑色） */
  --primary: oklch(0.45 0.31 264.05); /* 主要按鈕顏色（藍色） */
  /* ... 更多顏色 */
}

.dark {
  --background: oklch(0 0 0);        /* 深色模式背景（黑色） */
  --foreground: oklch(0.98 0 0);     /* 深色模式文字（白色） */
  /* ... 更多顏色 */
}
```

**簡單改法**（不用理解 oklch）：
如果想改主題色，找到 `--primary` 這一行，改成你喜歡的顏色。

---

### 7. 字體大小
**檔案位置**：`client/src/index.css`

找到這一段（大約第 190 行）：

```css
body {
  font-size: 17px;           /* ← 改大或改小 */
  line-height: 1.47;
}
```

**建議值**：
- 小字：`15px`
- 標準：`17px`（iPhone 預設）
- 大字：`19px`

## 📂 資料夾和檔案名稱

### 8. 專案資料夾名稱
你可以把整個 `iphone-notes-pwa` 資料夾改名成任何你想要的名稱，例如：
- `我的筆記本`
- `備忘錄備份`
- `個人記事`

**注意**：資料夾名稱可以用中文，但如果要上傳到 GitHub，建議用英文或拼音。

---

### 9. 檔案名稱
以下檔案名稱**可以改**（不影響功能）：
- `GOOGLE_SCRIPT_SETUP.md` → `Google試算表設定教學.md`
- `DEPLOY_TO_GITHUB.md` → `GitHub部署教學.md`
- `HOW_TO_DOWNLOAD.md` → `下載說明.md`

以下檔案名稱**不要改**（會影響程式運作）：
- `package.json`
- `vite.config.ts`
- `client/src/App.tsx`
- `client/src/pages/*.tsx`（所有程式檔案）

---

## 🗂️ 資料庫內容

### 10. 筆記內容
所有筆記的標題和內容都儲存在瀏覽器的 IndexedDB 中，**完全可以用中文**！

你可以：
- 用中文寫標題
- 用中文寫內容
- 建立中文名稱的資料夾
- 建立中文名稱的標籤

**完全沒有限制！**

---

### 11. 匯出的 JSON 檔案
當你匯出筆記時，JSON 檔案裡的內容也是中文的：

```json
{

    {
      "標題": "我的第一則筆記",

      "資料夾": "工作",
      "標籤": "重要, 待辦"
    }
  ]
}
```

## 📝 修改建議

### ✅ 可以放心改的地方
1. 所有中文文字（按鈕、提示訊息）
2. 顏色代碼（`#RRGGBB` 格式）
3. 字體大小（`font-size`）
4. emoji 圖示
5. 資料夾和標籤的預設設定
6. 說明文件的內容

### ⚠️ 小心修改的地方
1. CSS 變數名稱（`--background` 等）
2. 程式碼的邏輯部分（`if`、`function` 等）
3. import 路徑

### ❌ 不要改的地方
1. `node_modules` 資料夾
2. `package.json` 的相依套件版本
3. `.tsx` 和 `.ts` 檔案的副檔名
4. 程式碼的結構（除非你懂 React）

---

## 🛠️ 修改後如何測試？

1. **儲存檔案**
2. **重新整理瀏覽器**（如果開發伺服器正在執行，會自動更新）
3. **檢查是否正常運作**

如果改壞了：
1. 按 `Ctrl+Z`（Windows）或 `Cmd+Z`（Mac）復原
2. 或重新下載 ZIP 檔案

## 💡 快速修改範例

### 範例 1：改成「我的日記本」
1. 打開 `client/public/manifest.json`
2. 改成：
   ```json
   {
     "name": "我的日記本",
     "short_name": "日記"
   }
   ```

### 範例 2：把「建立第一則筆記」改成「開始寫日記」
1. 打開 `client/src/pages/NotesLocal.tsx`
2. 找到「建立第一則筆記」
3. 改成「開始寫日記」
4. 儲存

### 範例 3：改變主題色為粉紅色
1. 打開 `client/src/index.css`
2. 找到 `--primary`
3. 改成 `--primary: oklch(0.7 0.2 350);`（粉紅色）

## 📞 需要幫助？

如果修改後出現問題：
1. 檢查是否有拼字錯誤
2. 確認引號和括號是否配對
3. 查看瀏覽器的開發者工具（F12）是否有錯誤訊息
4. 重新下載原始 ZIP 檔案重新開始
